fix(canvas): stop worktree label from snapping back on drop

The label drag-drop path reset the imperative transform before the store
update propagated, so the label painted one frame at the old anchor
before re-rendering to the new position. Also the zoom fade was tied to
an opacity transition that never actually played (the element unmounts
at opacity 0), leaving a ghostly half-rendered band as you crossed the
threshold. Flush the store commit with flushSync so left/top updates
before the transform reset, and drop the fade gradient entirely so the
label is a clean on/off at HUD_THRESHOLD.
